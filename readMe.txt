这个project是用python写的数字1-9手写体的识别算法。
并且我上传了俩个训练文本trn_img.npy和dev_img.npy和一个测试文本，在trn_img.npy 中有10000行数据，每行数据有784列，每行代表一个手写体的数字，trn_lbl.npy 是他的标签，所以trn_lbl.npy 是一个维度为10000的向量，用来表示trn_img.npy每行所代表的数字。dev_img.npy 和dev_lbl.npy同上。
这个py文件里，写了俩个识别算法。
第一个就是最简单的取平均数法，即找到所有手写体为“1”的行，然后相加取平均得到“1”的代表向量，同样的这个向量有784列，然后对023456789分别做同样的事情。然后可以得到从0-9，10个有784列的代表向量。通过比较预测数据和这10个代表向量就可以预测手写体的数字。
第二个就是对KNN算法的模拟。
同时在文件中，还实现了acp算法，可以用这个算法实现数据特征量的提取，然后利用这些压缩的数据进行预测，可以大大的减小预测时间。
